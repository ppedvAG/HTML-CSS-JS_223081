Emmet

Was ist ein Emmet?

Kurzform um HTML zu schreiben. Schneller und einfacher als alles selbst einzutippen. Früher bekannt als Zen-Coding.

Wie funktioniert es?

Ich schreibe den Elements-Namen und drücke tab

Stellt auch weitere Funktionalität bereit wie das einsetzen von Id´s und Klassen

Emmet-Syntax

  • elementName + Tab => Erstellt öffnenden und (falls nötig) schließenden Tag
  • elementName.classname + Tab => Erstellt Tag mit entsprechender Klasse
  • elementName#idName + Tab => Erstellt den Tag mit entsprechender ID
  • element1>element2 + Tab => Erstellt element2 als Child von Element1
  • element1+element2 + Tab => Erstellt element1 und element2 als Siblings, also auf der selben Stufe
  • element1>element2^element3 + Tab => element 3 wird auf der selben Ebene wie Element1 erstellt
  • element1*n + Tab => Erstellt element1 n-mal
  • element1>(element2+element3)*4 + Tab => Würde viermal element2 benachbart von element3 innerhalb von element1 erstellen
  • element1[attribut=wert] + Tab => Erstellt das Element mit dem entsprechenden Attribut
  • element1#$Test + Tab => Stellt uns Nummerierung zur Verfügung, meistens in Kombi mit Multiplikator benutzt
  • element1#Test$@n*3 + Tab => Erstellt element1 3x mit Nummerierung und n als Startzahl. Vor dem n kann noch ein Minus stehen, dann würde n die Endzahl sein. Steht das Minus alleine wird absteigend nummeriert.
  • element1{Text} + Tab => Erstellt element1 mit dem entsprechenden Text
  • Bei lorem kann ich direkt hinter lorem eine Zahl schreiben, also ohne das Mal-Zeichen, und dann werden mir entsprechend viele Wörter generiert.

Article & Section

Können beliebig getauscht werden. Stehen beide für eigenständige Teile der Seite, die Informationen übermitteln. Sollten im Idealfall ein eigenes Heading haben. Können ineinander verschachtelt werden. Sind etwas besser für SEO als ein div, da sie ihre Funktion im Namen übermitteln. Die div hingegen gilt als leerer Tag. Gelten als inklusiver, da sie Screenreadern ihre Arbeit erleichtern.

Header & Footer

Können entweder als Kopf- und Fußzeile unsere gesamte Seite dienen, oder in Article und Section verwendet werden um zusätzliche Infos zu übermitteln

Nav

Ist für die Navigationsleiste gedacht. Sollte nur die Navigationsleiste enthalten

Aside Tag

Ist für interessante Infos innerhalb eines Textes gedacht, oder Bilder und weiterführende Infos.

Der Picture-Tag

Der picture-Tag bietet uns eine alternative zum img-Tag. Der Tag ist responsive. Der Tag enthält mindestens einen source-Tag und genau einmal den Img-Tag. Beim source-Tag gibt es noch das media-Attribut mit dem wir abfragen können wie groß der Bildschirm ist. Der img-Tag wird als Fallback benutzt, falls keiner der source-Tags passt oder der Browser den picture-Tag nicht unterstützt

Der map-Tag

Der map-Tag bietet uns die Möglichkeit ein Bild in mehrere Bereiche einzuteilen und diese Bereiche zu verschiedenen Links zu leiten. Dabei müssen wir aufpassen, dass wir die richtigen attribute verwenden, da das Bild selber auch noch das Attribut usemap="#nameDerMap" enthalten muss. Der map-Tag enthält mehrere area-Tags. Diese haben folgende Attribute:

  • shape-Attribut, kann folgende Werte haben:
    • rect, Viereckiger Bereich
    • circle, Runder Bereich
    • polygon, Vieleck
    • default, => selbe wie rect(Viereckiger Bereich)
  • coords, verhalten sich je nach shape unterschiedlich:
    • Rect: coords="x1, y1(obere linke Eck),x2,y2(rechtere untere Eck)"
    • Circle: coords="x1,y1(Mittelpunkt des Kreises), r(Radius)"
    • Polygon: coords="x1,y1(erstes Eck), x2, y2(zweite Eck)....."
  • href, Link wo bei Klick hingeführt wird
  • alt, Text der angezeigt wird wenn ich über dem Bereich hovere
